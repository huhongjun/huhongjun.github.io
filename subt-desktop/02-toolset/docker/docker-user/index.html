    backup¶

    传输本地镜像¶

    通过SSH传输Docker镜像，即时压缩内容：

    docker save | bzip2 | \ ssh user@host 'bunzip2 | docker load' 放在pv管道中间以查看传输的进展情况也是一个好主意：

    docker save | bzip2 | pv | \ ssh user@host 'bunzip2 | docker load'

    docker commit¶

    docker commit [OPTIONS] CONTAINER [REPOSITORY[:TAG]]

    OPTIONS说明：

    -a :提交的镜像作者； -c :使用Dockerfile指令来创建镜像； -m :提交时的说明文字； -p :在commit时，将容器暂停。

    docker commit -a "huhongjun" -m "conda pyexam" 16a966310601 docker.lan:9005/conda/miniconda3:v2

    AWS EC2 主机docker api远程访问(portainer)¶

    简介¶

    ssh隧道转发端口；本地主机端口 =》远程主机端口； portainer: 配置本地主机端口，但实际访问的是远程主机端口;

    实现案例 - ds3617xs-hd¶

    预备条件¶

    ssh登录

    1
2
    sudo chmod 600 /volume1/ARun/ssh/aws-ohio-ubuntu.pem
ssh -i /volume1/ARun/ssh/aws-ohio-ubuntu.pem ubuntu@3.14.138.219

    docker api开放

    1
    远程主机docker api设置为可以远程访问，默认是不可以。

    端口转发¶

    实现：本地端口(ds3617xs-hd的9527) =》[中转主机3.14.138.219] =》远程端口（远程主机的2375）

    手动

    1
2
3
    ssh -L 192.168.1.172:9527:127.0.0.1:2375 -i /volume1/ARun/ssh/aws-ohio-ubuntu.pem ubuntu@3.14.138.219 -N
ps ef|grep ssh
kill pid

    系统任务

    1
2
3
4
5
6
    ssh -L 192.168.1.172:9527:127.0.0.1:2375 -i /volume1/ARun/ssh/aws-ohio-ubuntu.pem ubuntu@3.14.138.219 -fN
