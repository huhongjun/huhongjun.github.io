DS3617xs管理指南

在ESXi环境下安装

ESXi新建虚拟机，要点： - StarWind Converter将启动硬盘synboot.vmdk转换为ESXi主机格式 - 加入现有硬盘，即下载的引导硬盘，sata 0:0 - 引导修改boot优先级，启动后选择菜单第三项（是特别针对esxi） - find.synology.com 或 Synology Assistant搜索设备 - 安装DSM_DS3617xs_23739.pat(Synology Assistant有显示23739)

日常使用

Synology容器 可以非常方便的进入终端，终端内类似XTerm，可以ping 主机局域网 新开终端，执行命令：sh，/bin/bash

MariaDB DS215j的数据库服务器remote账号特为远程登录 D:\ARun\Navicat 用remote账号可以访问

Tips

esxi和vmware workstation采用不同的硬盘格式。 可用StarWind Converter转换硬盘格式。

如何为虚拟机手动启用 CBT 启用 VMware 更改块跟踪 (CBT) 以便仅备份自上次备份时间以来更改的块，大幅减少传输的数据大小和备份时间。

本文章将会引导您完成在 VMware vSphere 中手动启用 CBT 的流程。请按照 VMware 说明进行操作并执行以下步骤。

若要在虚拟机中启用 CBT： 关闭虚拟机。如果不关闭虚拟机电源，您将无法成功完成设置。 右键单击虚拟机，然后单击编辑设置。 单击虚拟机选项选项卡。 单击高级，然后单击配置参数旁的编辑配置。即会打开“配置参数”对话框。 单击添加参数，添加 ctkEnabled 参数，并将其值设置为 true。 单击添加参数，添加 scsi0:0.ctkEnabled，并将其值设置为 true。 注： 每当将硬盘添加到虚拟机时，都会分配给硬盘一个 SCSI 设备。scsi0:0.ctkEnabled 中的 scsi0:0 代表 SCSI 设备，而其他设备的显示类似于 scsi0:1 或 scsi 1:1。 可以对每个硬盘分别启用或禁用 CBT。 开启虚拟机。 在虚拟机的主目录中，验证每个已启用 CBT 的硬盘中是否有 vmname-ctk.vmdk 文件。 以上说明参考此网站：VMware KB 1031873。在 VMware vSphere 的不同版本中可能会有所变化。您可以在 VMware KB 1031873 找到有关在 VMware 虚拟机中启用 CBT 的更多最新详细信息。

启用 CBT 之后，第一个备份仍会是完整备份，后续备份会是增量备份。
