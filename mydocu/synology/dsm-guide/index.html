DSM 非官方管理员手册

系统信息

cd /volume1/@docker sudo -i 登录只能admin admin/*，要用root需先设置

scp能上传文件，sftp好像不能

别名定义

echo "alias cddocker='cd /volume1/NFS-mount/ds215j/00-work/ws_docker'">>~/.profile

共享文件夹

/volume1/Backup

dsm ssh 免密登录

官方方案

官方参考

cd ~/.ssh rm authorized_keys cat id_rsa.pub >> authorized_keys

cd ~ chmod 711 . chmod 711 .ssh chmod 711 .ssh/authorized_keys chown -R admin .ssh/

网上方案

参考资料

// dsm开启个人家目录

cd /var/services/homes chmod 755 ./admin // 对于普通 linux 来说，这步就不需要了 chmod 700 ~/.ssh chmod 600 ~/.ssh/authorized_keys sudo vim /etc/ssh/sshd_config

```
RSAAuthentication yes
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
```

sshd启动失败还原

Log in as root (using the admin password) to do the following -

cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak cp /etc.defaults/ssh/sshd_config /etc/ssh/sshd_config sudo synoservicectl --restart sshd 或 dsm网页中先关后开ssh

配置本机快速登录

.ssh目录下config文件

```
Host ds3617xs-hd
    HostName 192.168.1.172
    User admin
    IdentityFile  C:\Users\huhon\.ssh\id_rsa
```

chmod 700
