NVIDIA

安装记录

cuda-toolkit 10.1安装成功，连带驱动好像升级到最新版了，并且cuda开发样例deviceQuery C++代码编译运行 ok

nvidia 使用

sudo watch -n 3 nvidia-smi nvcc -V // 需要安装cuda_toolkit，是CUDA编译器 sudo docker run --runtime=nvidia --rm nvidia/cuda nvidia-smi

TensorFlow、Python、CUDA和显卡驱动的版本关系

Dell G7 7588的NVIDIA私有驱动为384.130（安装操作系统后系统检测到的），在Linux上可以支持CUDA 9.0.76

conda安装 TF1.9=》然后重新安装1.10、1.11、1.12时CUDA不会自动升级，保持9.0.76版本，但如果新安装1.10以上会自动安装CUDA 9.2以上。

conda新安装TF1.8 只会自动安装CUDA 8版本

Windows 10 支持最新的NVIDIA驱动，而且可以双显卡同时使用； Ubuntu同一时刻只能激活一个显卡的驱动;

linux下NVIDIA X-config可以切换显卡

切换到Intel显卡，logout重登录即可 切换到NVIDIA显卡，logout重登陆，切换失败 Ctrl+Alt+F1重装显卡驱动再重启成功，也许切换后重启OS也能成功 Ctrl+Alt+F1 中文乱码问题=》直接使用英文版可以避免一些问题

Nvidia驱动安装

  • 离线安装

去NVIDIA驱动下载下载离线安装包，如：NVIDIA-Linux-x86_64-410.78.run

service lightdm stop chmod +x NVIDIA-Linux-x86_64-410.78.run ./NVIDIA-Linux-x86_64-410.78.run –no-opengl-files

  • 在线安装 sudo service lightdm stop sudo apt-get install nvidia-384 sudo service lightdm start =》 实际上需要操作系统重启

终端执行nvidia-smi可以查看nvidia显卡驱动版本和显卡硬件信息、内存使用情况

附录

GPU =>计算能力达48X0.95 = 45.6 GFLOPS CPU =>计算能力打4X2.5G = 10 GFLOPS CUDA只有nvidia显卡支持
